| D25 | A **compile failure during `run_tests`/Maven** (no Surefire XML produced) returns the operational error **`REPORT_NOT_PRODUCED`** (+ hint "run `build`"), raw compiler output retained in the `handle` ‚Äî **not** a `ContainerFinding`. The `build` verb (later PRD) owns compile-error ‚Üí `file:line` parsing. | The schema normalizes **reports**; when no report exists there is nothing to fold. Folding it as a finding would require **stdout scraping** of compiler output, which D8 rejects. The asymmetry vs Go (ADR-0007 rule 4 folds `build-fail`‚ÜíERRORED **as a finding**) is **report-format-driven**: `go test -json` emits structured build-fail events; Surefire XML is simply absent on a compile failure. Does not contradict ADR-0007 rule 2 (the ERRORED discriminator applies when a finding exists; here there is none). |
| D26 | PRD-1 pom-wiring locked: **JUnit 6.0.3** (inherit BOM, no override), **JaCoCo deferred** (TDD drives coverage; Java 25/GraalVM edges; add when a coverage gate is a real requirement), base package + groupId **`dev.nobash`**, artifactId **`no-bash-mcp`**. `Envelope.manager="mvn"`; `NormalizedRun.tool="surefire"` (the Reporter ‚Äî see CONTEXT.md). | KISS / YAGNI for the tracer; confirms the DESIGN ¬ß10 baseline and the ¬ß3 package proposal. |
| D27 | **Report freshness by construction (anti-false-green, in PRD-1).** Inject a **unique per-run reports directory** into the `ExecSpec` (`-Dsurefire.reportsDirectory=<fresh tmp>`, an MCP-controlled value like the injected `-Dtest=`, **not** an agent free-flag). The dir is empty before exec, so any `*.xml` in it is **necessarily** from this run; an **empty** dir after exec ‚Üí `REPORT_NOT_PRODUCED` (D25). **Supersedes** the original `mtime >= start` gate. | Surefire does **not** wipe `target/surefire-reports/` between runs (only `mvn clean` does), so a glob of the default dir would read **stale PASSED** reports and false-green (the exact G5 back door ‚Äî certain on compile-fail; also bites deselected-test / rerun). The first-cut `mtime >= start` gate was found, in adversarial review, to reopen the hole at coarse filesystem mtime granularity (1‚Äì2 s, /mnt DrvFs) where a stale file's truncated mtime equals `start`. A fresh per-run dir removes the whole class ‚Äî no wall-clock comparator, no granularity hazard, no workspace mutation (writes to tmp), and the freshness test needs no injected clock. Rejected: deleting the default dir pre-run (mutates workspace; loses evidence if exec never launches) and `mvn clean test` (full rebuild kills the inner loop). |
| D28 | **Process-exit is a failure floor at the envelope.** Envelope `ok = NormalizedRun.ok() && exitCode==0 && !timedOut && executedTests>0` (the `executedTests>0` term is D29). The frozen domain `NormalizedRun.ok()` stays **findings-only** (ADR-0007, unchanged); the process-signal dimension lives in the **application** (verb/dispatch) layer. A non-zero exit with all-PASSED (or stale) findings is **NOT** ok. | DESIGN ¬ß2 requires production to "also treat a non-zero process exit as a failure signal." Keeping it out of the domain record honors the ADR-0007 freeze while closing a second false-green angle (a post-test plugin failure / fork crash that exits non-zero after green XML is written). |
| D29 | **Positive-evidence floor: `executedTests > 0` + `NO_TESTS_RUN`.** All three earlier guards (container-aware `ok()`, freshness D27, exit-floor D28) key on **negative** evidence; none asserts tests actually ran, so a **fresh report with zero executed `<testcase>`** (no test matched, an all-`@Disabled` suite, an empty leaf module) + exit 0 vacuously greens (`noneMatch` on an empty stream is `true`). Add a **completeness term** to the envelope floor ‚Äî `executedTests = passed + failed + errored` (**excludes** `SKIPPED`) ‚Äî and a **new operational error `NO_TESTS_RUN`** (+ hint) when a fresh report exists but `executedTests == 0`. | Closes the vacuous-green class found in adversarial review. Distinct from `REPORT_NOT_PRODUCED` (no report at all) ‚Äî here a report exists but is empty. Lives in the application floor, so the frozen `NormalizedRun.ok()` is untouched. An all-`SKIPPED` run therefore does **not** silently green. |
| D30 | **Agent-facing envelope field = `failures[]` (carries `Finding`); domain record field = `NormalizedRun.findings`.** The test-failure Envelope exposes a top-level **`failures[]`** array (the established agent-facing term ‚Äî CONTEXT.md, tool-catalog.md, DESIGN ¬ß6), each element a `Finding` serialized with the frozen `@JsonTypeInfo(use=NAME, property="kind")` discriminator so the agent branches `TestFinding` vs `ContainerFinding` on `kind`. The frozen **domain** field stays `NormalizedRun.findings` (ADR-0007). | Resolves a corpus naming ambiguity (`failures[]` at the envelope layer vs `findings` in the frozen record) so the `jq -e` acceptance assertions and the agent's branch on `kind` are authorable. No frozen field is renamed; only the envelope‚Üîdomain mapping is pinned. |

## PRD-2 ‚Äî "remove-Bash-on-self" (grill 2026-06-04, in progress)

Decisions for the next phase: **v1 catalog completion toward the product thesis**, not the PRD-1
architecture. Scope per D32. Status: grill in progress, **not yet implemented**.

| # | Decision | Rationale |
|---|---|---|
| D31 | **PRD-2 sequencing axis = value-first ("remove-Bash-on-self"), not risk-first.** PRD-1 already proved the entire foundation (frozen schema ADR-0007, dispatch flow, `Envelope`, run-cache, `get_log`, `CommandExecutorPort`); everything remaining is **additive/leaf** ‚Äî Node/Go are more normalizers behind a proven port, forge is a self-contained 2nd port, native is terminal packaging. **Nothing foundational-risky remains**, so optimize for the product thesis instead of de-risking. | The product delivers **zero** value until the Bash permission can actually be removed; `run_tests` alone does not get there. With no risky base left to de-risk-before-building-atop, the highest-value next step is the smallest slice that realizes the thesis and unlocks dogfooding-on-self (which surfaces real empirical risk). |
| D32 | **PRD-2 scope = minimal dev-loop: git read-only (5 verbs) + Maven `build` + bootstrap skill.** Removes Bash for `edit ‚Üí build ‚Üí test ‚Üí inspect-git`. **Out of scope:** `gh` and **mutating** git stay on the **transitional dangerous-git deny-list** (D15/D16) ‚Äî Bash-for-git-mutation persists until post-v1 native mutating-git verbs. Forge (`pr_*`) ‚Üí PRD-3; `install`/`lint`/`run_task` ‚Üí later. | KISS/YAGNI: the smallest slice that delivers the remove-Bash thesis for *this repo's own* loop. Forge `pr_*` is read-only and does not cover issue/PR **create** (what `gh`/orchestrate use), so it does not help remove Bash from the dev loop ‚Äî it earns its own PRD. `install` is vacuous for Maven (D21); `run_task` is the highest-security verb (G14) ‚Äî both add surface without advancing the thesis. |
| D33 | **`build` compile-error output = new build-specific `CompileDiagnostic{file, line, col, severity, message}`, carried as envelope `diagnostics[]`** (distinct from test `failures[]`), parsed from maven-compiler-plugin `[ERROR] <file>:[<line>,<col>] <msg>` console lines; full compiler output retained via `handle`/`get_log`. The frozen ADR-0007 test schema (`Finding`/`SourceRef`) is **untouched**. ‚Üí candidate **ADR-0009**. | A compile diagnostic has **no test identity** and no test `Outcome` (PASSED/SKIPPED are meaningless), and frozen `SourceRef` carries **no column** ‚Äî folding it into `ContainerFinding(FILE)` would force `outcome=ERRORED`, drop the column, and stretch a schema ADR-0007 declared to be about *test results*. Console-parse is unavoidable (compile failures produce **no report-file**, unlike tests), but `file:[line,col]` coordinates are **locale-independent** (only the message text is localized ‚Üí passed through as untrusted content), so D8's "no fragile stdout scraping" concern is contained. Rejected: reuse `ContainerFinding` (above); tracer-with-no-parser (loses the `file:line` differential that justifies `build` over removed Bash). |
| D34 | **git read-only output = rich per-verb schemas parsed from `git --porcelain=v2` / `--format=`.** `git_status`/`git_branch` ‚Üí small inline structured shapes; `git_log` ‚Üí capped commit list (`sha`, `short`, `author`, `dateIso`, `subject`; default last-N) with full body via `git_show`; `git_diff`/`git_show` ‚Üí inline `files[]` (`path`, `added`, `deleted`, `status`) + the patch behind a `handle`/`get_log` (consistent with the already-decided `git_diff` handle reuse). `manager` is null for git verbs (CONTEXT.md). | Sufficiency-first + token-efficient + locale-stable: porcelain / `--format` is git's machine contract ‚Äî the project's "parse the machine format, never scrape stdout" rule (D8) applied to git. The highest-volume verbs (`git_log`/`git_diff`, 773-call evidence) get a handle boundary so large patches never flood the envelope. Rejected: thin-structured (agent re-parses author/date/ahead-behind ‚Üí weakens sufficiency) and passthrough-porcelain (pushes all parsing back to the agent ‚Üí contradicts agent-first). |
| D35 | **Transitional deny-list posture = "block destruction only."** The bootstrap's dangerous-git deny-list **denies** irreversible / history-rewriting / working-tree-loss ops and **allows** forward-progress git. DENY: `push --force`/`-f`/`--force-with-lease`, `push --delete`/`push :ref`, `reset --hard`, `rebase`, `commit --amend`, `filter-branch`/`filter-repo`, `clean -f`/`-fd`/`-fdx`, `checkout`/`restore -- <path>`, `checkout -f`, `branch -D`/`-d`, `tag -d`, `update-ref -d`, `stash drop`/`clear`, `reflog expire`, `gc --prune=now`, `prune`, `worktree remove --force`, `submodule deinit`. ALLOW: `add`, `commit`, `push` (plain), `branch <new>`, `switch`/`checkout <branch>`, `merge --ff`, `fetch`, `pull`. | Goal = the agent cannot **destroy** the repo, yet keeps its `edit ‚Üí commit ‚Üí push` loop (required for dogfooding-on-self). Accepts the **known G6 fragility** ‚Äî a denylist cannot catch `git -c <config-injection>`, aliases, or command-chaining ‚Äî because it is an explicitly **transitional bridge**; the real guarantee arrives when mutating-git becomes allowlist MCP verbs (post-v1 roadmap). Rejected: also-block-push (breaks this repo's push-driven flow, defers publish to a human) and block-all-mutation (agent loses the entire git write loop, contradicting "remove Bash without losing the loop"). |
