# Decision Log

Each row is a decision locked during the grill, with its rationale. Pillars are `P*`, gotchas `G*`.

| # | Decision | Rationale |
|---|---|---|
| D1 | Scope = command execution + (parked) safe Unix utilities | Read/Write/Edit/Grep/Glob already exist outside Bash; execution is Bash's unique territory. |
| D2 | Guarantee = "no agent-composed novel commands" | Honest & defensible; "zero dangerous code" needs a sandbox (G2). |
| D3 | Mechanism = argv via `ProcessBuilder`, never shell | Shell-string sanitization is a minefield (G1). |
| D4 | Flags = allowlist per operation, unknown dropped | Free flags = composition; denylist leaks (P3). |
| D5 | Catalog = logical verbs, one MCP | Small constant surface; dissolves per-repo tool filtering. |
| D6 | v1 verbs: `run_tests`(+target), `build`, `install`, `lint`, `run_task`, `describe_project`, `dependencies`, `get_log`, git read-only | Covers the implementation loop; evidence-driven additions. |
| D7 | v1 ecosystems = Maven (JVM) + Node (npm, pnpm, yarn) + Go; **Gradle deferred** | Three *dissimilar* report formats (JUnit XML, `jest --json`, `go test -json`) maximize schema de-risk and all have real local evidence. Gradle shares Maven's JUnit-XML parser → low marginal de-risk, deferred (G11). |
| D8 | Output = report-file parse → single normalized schema | Deterministic, locale/color-immune; stdout scraping is fragile. |
| D9 | Sufficiency-first / noise-truncated; operational errors = `code` + `hint` | One round-trip to actionable (P4). |
| D10 | Policy compiled-in; project config tunes non-sensitive knobs only | Guarantee survives agent repo-write (P8). |
| D11 | Native image decoupled: JVM core, native at release | Justified by distribution + footprint, not startup (G3). |
| D12 | Global / stateless install; `path` per verb; `timeout` + cap + process-tree kill | Works installed globally; no hangs. |
| D13 | Validation = JSpecify + Jakarta + programmatic guards | JSpecify is nullness-only; security boundary stays in code (G4). |
| D14 | Outbound content neutralized + marked untrusted | Confused-deputy / prompt-injection defense (P9). |
| D15 | Bootstrap skill (thin) writes MCP config + transitional git deny-list | Makes "remove Bash" deployable. |
| D16 | Guardrail mechanism = declarative deny-list, harness-agnostic adapter | Hook is bash + regex-on-string (G7); detect the harness. |
| D17 | `get_log` + `handle` for non-lossy token efficiency | Directly kills the #1 / P1 waste patterns (G5). |
| D18 | git read-only pulled into v1 | 773-call evidence; cheap ecosystem-agnostic adapter. |
| D19 | `run_task` = opt-in allowlist, fail-closed (human-authored) | Composition-safe ≠ consequence-safe (G14); `deploy:prod` is project-defined yet catastrophic. |
| D20 | Baseline = Micronaut **5.0.0** + Java **25**; Micronaut MCP **1.0.0** (MCP Java SDK 1.1.2) | Verified against the 5.0.0 platform release: Micronaut MCP 1.0.0 ships GA in the 5.0 platform with STDIO → 5/Java 25 is a GA, compatible foundation. |
| D21 | Preflight (`DEPS_NOT_INSTALLED`) seam is **vacuous for Maven**: kept in the dispatch flow but non-applicable (pass-through, never raised) for the Maven adapter. | Maven has no lockfile / `node_modules`-absence analog — deps resolve from `~/.m2` on demand during the build, and any real check would have to run Maven (defeating "preflight before exec"). Honest consequence: the PRD-1 tracer exercises every architectural seam **except** preflight, which is first genuinely exercised by the Node/Go PRD (npm `node_modules` absent / lockfile out-of-sync). |
| D22 | `RESOURCE_BUSY` lock key (ADR-0005 "per resolved target") = **canonical absolute path of the module directory** (`realpath(moduleDir)`, the dir holding the `pom.xml` whose `target/` is written). A target selector (class/method) does **not** change the key — concurrency is per module, not per test. | Deterministic, requires no pom parse to acquire the lock, and `realpath` collapses path aliases (`svc/../svc`) so no false free-pass. Rejected: `groupId:artifactId` (two distinct checkouts of the same artifact would false-collide; needs a pom parse) and the `target/` output dir (pom-configurable, equivalent to the module path but costs an effective-pom resolve). Extends naturally to multi-module (each module dir = its own key). |
| D23 | Ecosystem verbs invoke the **trusted system manager** (`mvn` on `PATH`, `.cmd`/`.bat` shim on Windows), **never** a repo-authored wrapper (`./mvnw`). Absent → `TOOL_NOT_INSTALLED`. See **[ADR-0008](../adr/0008-trusted-manager-launcher-not-repo-wrapper.md)**. | The agent can rewrite `./mvnw` (P8: it has repo write access); invoking it would make `run_tests` an agent-composed arbitrary-command vector (P1/P2). Trade: a repo-pinned manager version is not honored (security > reproducibility). |
| D24 | PRD-1 ships a top-level **`README.md`** generated via `/code-documentation:doc-generate` as the **final issue**, run only **after the tracer is green**. It documents **only the shipped surface** (`run_tests`/Maven, install, MCP registration, remove-Bash), carries a "v1 in progress" banner, and **points to** `DESIGN.md`/`CONTEXT.md`/`docs/` rather than duplicating them. doc-generate is **forbidden from regenerating the hand-authored corpus** (DESIGN.md, ADRs, CONTEXT.md, docs/design). One-time generation; CI doc-generation + diagram/API suite deferred (YAGNI, post-v1). | The repo had no entry-point README. Generating from the real artifact (not up front) avoids false precision (documenting features that don't exist); the points-don't-duplicate rule preserves the documentation-first source-of-truth split (README = orientation; DESIGN.md = architecture; CONTEXT.md = glossary). |
| D25 | A **compile failure during `run_tests`/Maven** (no Surefire XML produced) returns the operational error **`REPORT_NOT_PRODUCED`** (+ hint "run `build`"), raw compiler output retained in the `handle` — **not** a `ContainerFinding`. The `build` verb (later PRD) owns compile-error → `file:line` parsing. | The schema normalizes **reports**; when no report exists there is nothing to fold. Folding it as a finding would require **stdout scraping** of compiler output, which D8 rejects. The asymmetry vs Go (ADR-0007 rule 4 folds `build-fail`→ERRORED **as a finding**) is **report-format-driven**: `go test -json` emits structured build-fail events; Surefire XML is simply absent on a compile failure. Does not contradict ADR-0007 rule 2 (the ERRORED discriminator applies when a finding exists; here there is none). |
| D26 | PRD-1 pom-wiring locked: **JUnit 6.0.3** (inherit BOM, no override), **JaCoCo deferred** (TDD drives coverage; Java 25/GraalVM edges; add when a coverage gate is a real requirement), base package + groupId **`dev.nobash`**, artifactId **`no-bash-mcp`**. `Envelope.manager="mvn"`; `NormalizedRun.tool="surefire"` (the Reporter — see CONTEXT.md). | KISS / YAGNI for the tracer; confirms the DESIGN §10 baseline and the §3 package proposal. |
| D27 | **Report-freshness gate (anti-false-green, in PRD-1).** Stamp `start` immediately before exec; the Surefire normalizer accepts **only** report files with `mtime >= start`. No fresh file → `REPORT_NOT_PRODUCED` (D25). | Surefire does **not** wipe `target/surefire-reports/` between runs (only `mvn clean` does). A glob of `*.xml` would read **stale PASSED** reports from a prior run and emit a success envelope — the exact G5 false-green, reintroduced via the back door (certain in the compile-fail case where no new XML is written; also bites deselected-test / rerun cases). Rejected: deleting the report dir pre-run (mutates the workspace; destroys prior evidence if exec never launches) and `mvn clean test` (full rebuild kills the incremental inner loop). `/tdd` must verify Surefire mtime behavior + filesystem mtime granularity. |
| D28 | **Process-exit is a failure floor at the envelope.** Envelope `ok = NormalizedRun.ok() && exitCode==0 && !timedOut`. The frozen domain `NormalizedRun.ok()` stays **findings-only** (ADR-0007, unchanged); the process-signal dimension lives in the **application** (verb/dispatch) layer. A non-zero exit with all-PASSED (or stale) findings is **NOT** ok. | DESIGN §2 requires production to "also treat a non-zero process exit as a failure signal." Keeping it out of the domain record honors the ADR-0007 freeze while closing a second false-green angle (a post-test plugin failure / fork crash that exits non-zero after green XML is written). |
